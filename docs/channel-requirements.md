# 要件定義: 投資データ検証チャンネル(仮称未定)

版: v1.0(2026-07-20)/ 根拠: docs/video-automation-research.md(§1〜9)

## 1. コンセプト

> 海外の一次データ(SEC EDGAR/FRED/CFTC)を取得し、日本株への影響を「過去データの検証」で可視化する非属人チャンネル。予想は一切しない。数字だけを見せる。

- 視聴者: 日本株・新NISA世代の個人投資家(中級寄り)
- 画作り: How Money Works型モーショングラフィック+Remotionチャートアニメ+「お金の3D可視化」を看板演出に
- 音声: VOICEVOX(キャラは制作時に選定、クレジット表記)

## 2. 投稿設計(二層+臨時)

| 種別 | 頻度 | 内容 | 自動化度 |
|---|---|---|---|
| Shorts「今朝の海外→日本株」 | 毎日(平日) | 金利・為替・SOX・先物の4枚グラフ定型 | ほぼ全自動(公開判断のみ人間) |
| 長尺「今週の海外マネー→日本株」 | 週1(日曜) | 週間検証+バズ企画(もしもシミュレーション/バーチャートレース/通説検証/暴落アーカイブ) | 半自動(台本チェック+検収) |
| イベント臨時便 | 月2〜3 | FOMC・米CPI・大型決算の翌日検証 | 半自動 |

## 3. 信頼設計(仕様として実装)

出典+取得日を画面内常時表示/概要欄に一次ソース全URL/分析コード・データをGitHub公開/訂正ポリシー(固定コメント+訂正一覧)/定時投稿/「投資助言ではない・売買推奨しない」明示

## 4. 規制チェックリスト

- 個別銘柄の売買推奨をしない(特に有料・会員限定では絶対に=投資助言業)
- アフィリエイトはPR表記+金融広告表現規制(最上級表現禁止)
- 量産型対策: 台本は検証データ固有の記述、素材使い回し抑制、1日3本未満

## 5. パイプライン(実装スコープ)

```
[P1] データ収集: EDGAR/FRED/COT/株価(yfinance) → SQLite
[P2] 分析: イベント×過去N回の日本市場反応を自動集計
[P3] 台本: LLMがJSON台本ドラフト → 人間チェック(検収UI or Markdown)
[P4] 音声: VOICEVOX ローカルAPI
[P5] 映像: Remotion(チャートアニメ・字幕・出典表示テンプレ)→ FFmpeg
[P6] 投稿: YouTube Data API(監査申請は初日に提出。通過まで private→手動公開)
[P7] 派生: 図解1枚のX投稿画像も同時生成
```

- コスト目標: 月1,000円台(LLM+電気代)。技術: TypeScript/Node中心(Remotionと統一)
- マイルストーン: V1=P1+P2+P5で「最初の長尺1本」を制作 → V2=Shorts日次自動化 → V3=API投稿+X連携

## 6. KPI(現実ライン)

- 3ヶ月: 動画15本・登録300人 / 6ヶ月: 登録1,000人(収益化ライン)/ 12ヶ月: 月1〜3万円+アフィリ開始
- 撤退基準ではなく学習基準: 6ヶ月で登録500人未満なら企画フォーマットを再設計(チャンネルは継続)
