# L2深掘り: 推し活管理アプリ — 競合分析・ユーザー実態・コンセプト案

調査日: 2026-07-17
調査方法: 2系統の並行調査(①推し活管理アプリの競合詳細、②推し活ユーザーの行動実態・ペイン)+プレイガイド接続可能性の個別確認

---

## 1. エグゼクティブサマリー

- 推し活市場は**約3.5兆円・人口1,384万人・年間平均支出約25.5万円**。Z世代女性の約8割に推しがいて、**20代は月3万円以上使う層が31%と全年代最多**。
- 競合調査の結論: **「申込→当落→入金→発券→遠征→収支」を一気通貫で扱うアプリは存在しない**。最大手Oshibana(100万DL)はデコ・記念日特化で管理機能が空白。ユーザーはスプシ+Trello+Googleカレンダー+紙手帳の3〜4ツール併用でしのいでいる。
- 最大のペインは**当落ライフサイクル管理**。入金期限は1秒過ぎても自動キャンセル+FCでの信用失墜(複数回でブラックリスト化の報告も)という金銭・信用の実害があり、「申込を登録すると期限通知が自動で飛ぶ」仕組み(ユーザー発案)はこのペインの中心に刺さる。
- 注意点: 「管理機能への課金」は前例が弱い(&Fanfunは無料のまま、個人開発系は広告止まり)。収益化はOshibanaが実証した**デコ・シェア体験への課金+アフィリエイト**を組み合わせる設計が必要。

## 2. 競合の機能カバレッジ(調査①の結論)

| 機能 | Oshibana | カレンダー系 | &Fanfun | 収支系(推しPay等) | 当落系(チケコレ等) | 紙手帳 |
|---|---|---|---|---|---|---|
| 現場・配信スケジュール | ✕ | ◎ | △ | △ | △ | ◯ |
| **当落管理(申込→結果→入金期日)** | ✕ | ✕ | △ | ✕ | ◎(小規模) | ◯(手書き) |
| 遠征(交通・宿・持ち物) | ✕ | ✕ | ◎ | ✕ | ✕ | △ |
| 収支・予算管理 | ✕ | △ | ✕ | ◎ | △ | △ |
| ウィジェット | ◎ | ◯ | ✕ | ◯ | ✕ | — |
| 規模 | 100万DL | 小〜中 | 1万DL | 中 | 小 | 大(書店流通) |
| 収益化 | ◎(サブスク月400円+物販+B2B) | 広告 | 未収益化 | 広告/微課金 | ほぼ無し | 買い切り |

### 競合から得た教訓

- **Oshibana**: 管理ではなく「デコ・推し色・記念日」で100万DL・評価4.83。課金はデコ素材サブスク(月400円/年3,800円)+物販+クリエイターRevShare。**「愛でる体験」に課金させている**
- **チケコレ/推し活ポケット**(個人開発の当落管理): 「チケット管理の理想形」と高評価だが小規模。最大の不満は**手入力の手間**
- **&Fanfun**(遠征特化・VC出資): 好評だが1万DL・未収益化。**単機能では規模も収益も出ない**
- **紙の推し活手帳**: 当落管理ページ標準装備の商品が毎年20種以上出る=**紙に印刷されたフォーマットが現在の「決定版」**。ただし「日程変更でぐちゃぐちゃになりアプリに移行した」という声も
- 不満の共通項: バグ・同期不安(Oshibana)、入力が面倒(チケコレ)、**機能が単発で横につながらない**(全般)

## 3. ユーザー実態(調査②の要点)

### お金
- 支出内訳トップ3: グッズ30.7%、チケット29.7%、遠征23.0%。遠征・グッズは93%が「1年前より支出増」
- 分布は二極化: 全体では月1万円未満が約7割だが、20代は月3万円超が31%
- 推し活貯金の文化が確立(専用口座・袋分け・「推しがテレビに出たら500円」ルール等)
- 使いすぎ後悔が最頻出の負の感情(「全種そろえないと気が済まない」)

### 当落(最重要ペイン)
- FC先行→各種先行→復活当選→一般の多段抽選。1ツアーで申込機会4〜6回×複数公演×複数名義=**申込が数十件に膨らむ**
- 重複当選が全滅につながるルールもあり「紙に書き出して自衛」が定番
- **入金忘れ=即キャンセル+信用失墜**。Trelloで「申込前/当落待ち/未入金/発券待ち」を管理する人、専用アプリを自作する人まで出ている
- 遠征は1回3〜6万円、同行者との立替精算がトラブルの種

### シェア文化(グロースの種)
- スクショ欲求が生まれる瞬間: **当選画面、戦利品並べ、祭壇・痛バ完成、年間支出/現場数の振り返り**
- 「愛を綴るのは手帳、期限とお金の実務はアプリ」で併用されている → 両方に寄せる設計が可能

### ペルソナ(要件定義のインプット)

1. **多ステ遠征社会人**(25歳・年間40万円級・複数名義): 当落ステータス管理・期限通知・遠征精算が刺さる
2. **身の丈推し活学生**(19歳・月5,000円〜1万円): 推し貯金の可視化・予算上限・デコ記録とシェアが刺さる
3. **在宅配信課金型**(22歳・VTuber/歌い手): 課金累積の可視化・配信カレンダー・「推しに使った総額」カードが刺さる

## 4. コンセプト案: 「推し活の"期限とお金"を全部引き受けるアプリ」(仮称: 当落ノート)

> 申込を登録した瞬間に、当落発表・入金締切・発券期限・FC更新の通知がぜんぶ自動でセットされる。当選したら遠征と予算に自動でつながる。

### コア機能(当落ライフサイクル管理)

```
申込登録 → [当落待ち] → 当選 → [入金待ち] → 入金済 → [発券待ち] → 参戦 → 記録
              ↓落選                ↑
           復活当選待ち      期限通知(前日・当日朝の2段階)
```

- **申込→通知の自動化**(ユーザー発案の中核機能):
  - Tier 1(MVP): プレイガイド別テンプレで申込を数タップ登録 → 当落発表日・入金期日・発券期限・一般発売日の通知が自動セット
  - Tier 2: アプリ内から申込ページを開く導線(**ぴあ・ローチケはアフィリエイト提携可能** = 申込導線自体が収益源)+申込確認メールの貼り付け/転送をAIで解析して自動登録
  - やらないこと: プレイガイドAPIは非公開のため申込の完全自動化はしない(規約違反・法的リスク)。「申込は公式、管理と通知はこのアプリ」の役割分担
- 複数名義・複数公演のマトリクス管理(名義×公演の当落表)
- 当選→遠征プラン(交通・宿・持ち物・同行者メモ)→予算への自動反映(**当落×収支の連動はどの競合もやっていない**)

### 収益設計(Oshibanaの教訓を反映)

| 収益源 | 内容 | 根拠 |
|---|---|---|
| Pro サブスク(月300〜480円/年2,900円) | 名義・推し数無制限、ウィジェット追加枠、シェアカードのデコ素材、年間まとめ | Oshibana月400円が通っている価格帯 |
| アフィリエイト | ぴあ・ローチケへの申込導線、遠征の交通・宿(じゃらん/楽天トラベル等) | 申込・遠征という行動の上流を押さえる立地 |
| リワード/バナー広告(無料版) | 控えめに。広告除去はProに含める | 日本のeCPMは世界トップ級 |

- 無料版で当落管理のコアは全部使える(管理機能に課金の壁を作ると前例通り失敗する)。課金は「もっと推せる・もっと映える」側に置く

### グロース設計(TikTok/X自走)

- **当選報告カード**: 当選の瞬間に「祝・当選」のスクショ映えカードを自動生成(推し色・公演名入り) → 当選報告はX文化として確立済みなので自然に露出
- **年間まとめカード**: 「今年の現場◯回・遠征◯回・総額◯円」のSpotify Wrapped型振り返り(12月に拡散が狙える)
- 檻(データ)が貯まるほど乗り換えコストが上がる=ワークフロー・ロックイン

### リスクと対策

| リスク | 対策 |
|---|---|
| 管理機能への支払い意欲が弱い | コアは無料、課金はデコ・シェア・無制限枠。アフィリエイトで補完 |
| チケット転売との誤認・悪用 | 転売助長機能(譲渡マッチング等)は持たない。個人の管理に限定 |
| プレイガイドの仕様変更(先行形態・期日) | テンプレはユーザー編集可能にし、マスタ依存を減らす |
| Oshibana等の機能拡張 | デコではなく「期限とお金」の実務に先に根を張る(Oshibanaは管理が空白のまま100万DL) |
| ジャンル差(名義文化はジャニーズ系中心等) | MVPはジャンル横断の共通ループ(申込→期限→収支)に絞り、用語はカスタマイズ可能に |

### 数値目標(1年目)

- 先行類似(チケコレ・推し活ポケット)が小規模に留まる理由=発見性と入力コスト → 当選報告カードのシェア導線で突破
- 目標: 1万DL・Pro転換3〜5%(300〜500人)×月400円 ≒ **月12〜20万円** + アフィリエイト
- リリースタイミング: ツアー当落が集中する時期(春・秋)+年間まとめ(12月)

## 5. G1(野球監督シム)との比較

| 観点 | L2: 推し活管理 | G1: 野球監督シム |
|---|---|---|
| 需要の質 | 実害のあるペイン(入金忘れ・使いすぎ) | 娯楽の空白(栄冠クロス撤退) |
| 収益の型 | サブスク+アフィリエイト(積み上げ型) | 広告+アイテム課金(ヒット型) |
| 開発規模 | 小さく出せる(CRUD+通知が中心) | 試合シミュレーション設計が重い |
| グロース | 当選報告カードのシェア自走 | 野球SNS/実況+季節スパイク |
| リスク | 支払い意欲の弱さ、単機能止まり | バランス設計、埋もれ |
| バイブコーディング適性 | ◎(Web技術の王道領域) | ◎(テキストシムなら) |

L2は「実装が軽く、失敗しても学びが残る」堅実型。G1は「当たれば大きいが設計勝負」の上振れ型。
