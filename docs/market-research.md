# 市場調査レポート:個人開発(バイブコーディング)で作る「役に立つ×マネタイズ可能」なアプリ

調査日: 2026-07-17
調査方法: Web調査(海外インディー開発者市場 / 日本市場 / 飽和領域・実現可能性の3系統を並行調査し統合)

---

## 1. エグゼクティブサマリー

- **勝ち筋は「巨大なアイデア」ではなく、支払い意欲の高いニッチ(B2B/バーティカル)× 日本語・日本の制度というローカルモート × サブスク課金**の組み合わせ。
- 汎用AIラッパー、ToDo/ポモドーロ/習慣トラッカー、AIヘッドショット・履歴書ビルダー等は**完全に飽和**しており避けるべき。
- 個人開発者にとって最も現実的なプラットフォームは **Webアプリ(Next.js + Supabase + Stripe)**。審査ゼロ・即日デプロイ・決済直結で、ランニングコストは月0〜50ドルに収まる。
- 日本市場では **2025〜2026年の制度変更(フリーランス新法、インボイス2割特例終了、電子帳簿保存法、中小受託取引適正化法)** がツール需要を構造的に生んでおり、海外製SaaSやクローンが参入しにくい。
- 現実的な収益目標: 1年目で月5〜15万円($500〜1K MRR)。これ自体が上位2割の成果であり、達成を分けるのは技術力ではなく「着手前の顧客理解・強気の価格設定・単一チャネルへの集中」。

## 2. 世界のインディー/マイクロSaaS市場の実態(2025-2026)

### 収益の分布(現実的なベンチマーク)

- 平均的なマイクロSaaSは約$1,735 MRR(利益率64%)。黒字プロダクトの中央値は約$4.2K MRR。収益化済みSaaSの44%が単独ファウンダー運営([Flowjam](https://www.flowjam.com/blog/27-micro-saas-examples-that-actually-print-money-in-2025))。
- **新規アプリの2年以内$1K MRR到達率は17.3%、$10K到達は4.6%**([RevenueCat State of Subscription Apps](https://www.revenuecat.com/state-of-subscription-apps))。
- Indie Hackers上でStripe連携し収益公開している937製品のうち54%は売上$0。1年目の中央値は月$1,000未満([BuildMVPFast](https://www.buildmvpfast.com/blog/quit-software-engineer-indie-hacking-35-career-transition-2026))。
- $1K MRR到達組と未達組を分ける要因: (1) 価格が安すぎる、(2) 獲得チャネルを同時に追いすぎる、(3) 買い手の確約なしに作り始める。

### 成功事例のパターン

| 事例 | 収益 | パターン |
|---|---|---|
| Pieter Levels (Photo AI等) | ポートフォリオ計 $3.1M+ ARR | ニッチAI×ソロ運営×複数プロダクト |
| Marc Lou (ShipFast等15製品) | 2025年 $1.03M | 小プロダクト積み重ね+相互送客 |
| Senja(テスティモニアル収集) | $1M ARR | B2Bニッチワークフロー |
| WideBundle(Shopifyアプリ) | $50K MRR | マーケットプレイス流通内蔵 |
| Habit Pixel(習慣トラッカー) | 8ヶ月で$1K MRR | 等身大のソロ事例 |

### マネタイズモデルの有効性

- サブスク採用アプリはモバイル全体の4%だが**収益の45.4%を占有**([Airbridge](https://www.airbridge.io/en/blog/subscription-vs-one-time-purchase-app))。
- 年額プランの12ヶ月リテンションは44.1%で月額(17.5%)の約2.5倍([RevenueCat](https://www.revenuecat.com/state-of-subscription-apps))。
- フリーミアム転換率の目安: 3〜5%で良好、平均5.6%([ChartMogul](https://chartmogul.com/reports/saas-conversion-report/))。
- AI機能を持つアプリは「基本サブスク+クレジット制(従量)」が標準化。安い定額+AI使い放題は原価倒れ&高チャーンの最悪構成。
- 広告モデルはソロSaaSではほぼ非主流(後述の日本のデータでも同様)。

### 流通・ユーザー獲得

- オーディエンスなしで再現性が高いのは **①コールドアウトリーチ+自分のネットワーク、②Reddit等のコミュニティ**(最初の100ユーザー中60がReddit経由の事例あり)。
- SEOは中期の主力(11〜100人目の顧客の30〜50%がSEO経由という報告)。
- Product Huntは飽和気味。マーケットプレイス(Shopify/Chrome Web Store等)は「流通内蔵」で有利。

### AI/バイブコーディングの影響

- 2026年、開発は「作るのが簡単、気づかれるのが困難」に逆転。薄いAIラッパーの60〜70%は売上ゼロ、90%が2026年末までに消えるとの予測([Market Clarity](https://mktclarity.com/blogs/news/ai-wrapper-market))。
- 防御性の源泉は ①流通・オーディエンス、②業務ワークフローへの深い組み込み、③利用で貯まる独自データ、④実務者しか知らないバーティカル知識 に移行。

## 3. 日本市場の動向とギャップ(2025-2026)

### 市場動向

- 日本のアプリ消費支出は世界第3位(2024年 約165億ドル)。**非ゲームが+23%成長し2025年にゲームを逆転**([Sensor Tower](https://sensortower.com/ja/blog/state-of-mobile-2025-JP))。
- 2025年上半期、金融系アプリはインストール+50%と突出([Adjust](https://www.adjust.com/blog/japan-app-trends-2025/))。
- 教育アプリ市場はCAGR 21.5%で成長予測([IMARC](https://www.imarcgroup.com/japan-education-apps-market))。AI英会話が特に好調。
- 介護DX市場は545.8億円(2024年度)で拡大中。

### 日本特有のギャップ(海外SaaS・クローンが埋められない領域)

- 中小企業の実態: 電話・FAX受発注が2割超、紙台帳・Excel顧客管理が約4割([東京商工会議所](https://www.tokyo-cci.or.jp/file.jsp?id=1205203))。
- **フリーランスは本業約462万人、広義1,303万人・経済規模20.3兆円**([Biz/Zine](https://bizzine.jp/article/detail/11359))。
- **個人事業主のクラウド会計利用率は38.3%に過ぎず、6割超が未利用**。「freeeは高い・多機能すぎる」層向けの軽量ツールに明確な空白([MM総研](https://www.m2ri.jp/release/detail.html?id=672))。
- 電子帳簿保存法「対応済み」は中小企業の19.8%。未対応理由の上位は「導入コストが高い」34.7%、「最適なシステムが見つからない」30.6%([ワークフロー総研](https://www.atled.jp/wfl/article/41389/))。
- 適格請求書の様式、源泉徴収、支払サイト、FAX併用など、**「日本の制度への完全準拠」自体が差別化**になる。

### 制度変更起点のチャンス(時限性のある追い風)

| 制度 | 時期 | 生まれる需要 |
|---|---|---|
| フリーランス新法 | 2024/11施行済 | 発注条件明示(3条書面)・60日以内支払の管理ツール |
| スマホ新法 | 2025/12全面施行 | 外部Web決済解禁→個人開発の収益性が構造的に改善 |
| 中小受託取引適正化法 | 2026/1施行 | 発注書面・支払期日管理(対象がクリエイティブ業務等に拡大) |
| **インボイス2割特例の終了** | **2026/9末を含む課税期間** | 免税事業者上がりの数百万事業者に税額シミュレーション・申告支援需要が集中。※令和8年度税制改正で個人限定の「3割特例」(2027年分・2028年分)が創設済み(成立済みの確定制度)で、制度変化は2029年まで継続 |
| カスハラ対策義務化 | 2026/10予定 | 記録・チェックリスト系ツール |

### 日本の個人開発者の収益実態

- AI×個人開発でApp Store 1位・月収250万円の事例、ニッチゲームで月100万円、8年継続で月20万円など。
- 広告(AdMob)型は「1日1万UUで月10万円」が目安で単価が低い。**月100万円超の事例はニッチ特化+サブスク/課金型に集中**。
- 日本人のサブスク許容価格: 最多価格帯は月1,000〜3,000円、次いで500〜1,000円。**個人向けは月300〜980円、事業者向けは月1,000〜3,000円がスイートスポット**([サブスク管理帳](https://subsuku.net/articles/subscription-average-spending))。
- ストア手数料: 年収益100万ドル以下は15%(Small Business Program)。スマホ新法により外部Web決済へのリンクアウトが解禁(Apple 15%/Google 20%+決済費約3%)。

## 4. 避けるべき飽和領域

- **汎用AIラッパー全般**: AIライティング、AI画像生成、AIチャットボット、AI議事録要約、AI SNSスケジューラーは完全飽和。90日チャーン平均65%(SaaS平均の約2倍)([Indie Hackers](https://www.indiehackers.com/post/the-uncomfortable-truth-about-ai-tool-pricing-in-2026-92944b6a4d))。
- **定番ユーティリティの正面突破**: ToDo・ポモドーロ・習慣トラッカーは確立プレイヤー+無数のクローンが競合。
- **AIヘッドショット・履歴書ビルダー**: 大手の無料バンドル化で価格崩壊が進行。
- 薄いWebラッパー型モバイルアプリはAppleガイドライン4.2でリジェクトされやすい。

## 5. 技術・プラットフォーム選定の示唆(バイブコーディング適性)

| プラットフォーム | 適性 | 理由 |
|---|---|---|
| **Webアプリ(Next.js+Supabase)** | ◎ 最推奨 | 審査ゼロ・即日デプロイ・Stripe直結(2.9%+30¢)。MVPが数日で立つ |
| Chrome拡張 | ○ | 数日でリリース可能。審査はモバイルより軽い |
| PWA | ○ | 審査なし・即時アップデート。ただしストア発見性ゼロ |
| LINEミニアプリ | △ | 日本の店舗系B2B2Cで有望だが、認証済みは個人だと審査が通りにくい |
| モバイル(Flutter/RN) | △ | 審査・IAP・OS追随の保守負担が最も重い |

### コスト構造

- ホスティング: Vercel + Supabase の無料枠で開始可、成長後も月$45程度。Cloudflare併用で抑制可能。
- LLM API: 2025→2026年で約80%下落。プロンプトキャッシュで最大90%減。AI機能は「あり」でも原価管理必須。

### バイブコーディングの典型的失敗と対策(必須チェックリスト)

vibe-codedアプリの65%に脆弱性・58%にクリティカルという調査あり([The New Stack](https://thenewstack.io/vibe-coding-could-cause-catastrophic-explosions-in-2026/))。Lovable製アプリの70%がSupabase RLS無効。

- Supabase RLSを「デフォルト拒否+ユーザー単位ポリシー」で必ず有効化([Supabase公式チェックリスト](https://supabase.com/blog/the-vibe-coding-master-checklist))
- 認証はマネージド(Supabase Auth / Clerk)。クライアント側のみの認可は禁止
- レート制限(Upstash Redis + ミドルウェア)
- リリース前に gitleaks + semgrep でシークレット/脆弱性監査

## 6. 結論: 狙うべき交差点

1. **市場**: フリーランス・小規模事業者の「制度対応×お金まわり」— 数百万人規模、支払い意欲が高く、大手(freee等)が重すぎる層が6割残っている。制度変更(2026/9 インボイス2割特例終了など)という時限性の追い風がある。
2. **形**: Webアプリ(Next.js + Supabase + Stripe)。月額500〜1,500円のサブスク+無料プラン。
3. **防御性**: 日本の制度・商習慣への深い準拠(ローカルモート)+ ユーザーの業務データが貯まるワークフロー組み込み。
4. **流通**: SEO(制度名での検索需要)+ X/コミュニティでのビルドインパブリック。
