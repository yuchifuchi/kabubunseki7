# ゲーム候補案と評価(エンタメ転換版)

[ゲーム市場調査レポート](./game-market-research.md) をもとに、野球・RPG系×アイテム課金で個人開発(バイブコーディング)が現実的な候補を5案に絞り評価した。

---

## 評価基準(各5点満点)

| 基準 | 内容 |
|---|---|
| 市場性 | 需要の証拠(検索需要・先行事例・空白) |
| 収益性 | アイテム課金の設計余地と実績ベースの期待値 |
| 競合 | 競合の少なさ・差別化余地(5=競合が弱い) |
| 開発適性 | バイブコーディング(Web技術中心)での実現しやすさ |
| 運営の軽さ | 1人で回せるか(5=作り切りに近い) |
| 追い風 | タイミング・季節性・トレンド |

## 候補G1: 高校野球監督シミュレーション(栄冠ナイン系)【推奨】

**架空の公立高校の監督となり、3年周回で全国制覇を目指すテキストベース育成シム**

- **需要の空白**: 公式スマホ版『栄冠クロス』が2026年3月にサ終し、スマホの本命が不在。「栄冠ナイン スマホ」系の検索需要は恒常的に残存
- **ゲームループ**: 選手スカウト→練習采配→試合(テキスト+簡易演出のシミュレーション)→世代交代の3年周回。栄冠クロスの敗因(ガチャ特待生・スタミナ制)を反面教師に、**「自由に周回できる買い切り的体験」を無料+アイテム課金で提供**
- **課金設計**(通貨なし・ガチャなし):
  - 引退選手を「殿堂(OB名鑑)」に永久保存するチケット(¥160)← 先行事例で収益10倍化が実証された「感情課金」の踏襲
  - 広告除去(¥980買い切り)/ ユニフォーム・校名エディット拡張(¥320)/ 周回引き継ぎスロット拡張(¥480)
  - リワード広告: スカウト再抽選、練習効率ブースト等
- **技術**: React/Next.js(Web) + Capacitorでモバイル化しストアIAP。試合シミュレーションはサーバー不要のクライアント完結設計で運営コスト最小化
- **リスク**: 「甲子園」の語は商標リスクがあるため「全国大会」等に言い換え。試合バランス調整(先行インディーの不満点「終盤の運ゲー」)に設計力が要る

| 5 | 4 | 4 | 4 | 4 | 5 | **26** |

## 候補G2: 野球×ローグライトデッキ構築(采配カードバトル)

**打順・采配・特訓を「カード」として毎試合デッキを組み替え、負けたら世代交代から再挑戦するローグライト**

- ローグライト/デッキ構築は「小規模でも高評価を得やすい鉄板ジャンル」(2026年Q1国産Steam分析)。野球との掛け算は前例が薄く独自性が高い
- 課金: 追加シナリオ・カードパック(中身固定の直接販売、ガチャではない)、広告除去
- リスク: デッキ構築のバランス設計は監督シムより難度が高い。野球ファンとローグライトファンの重なりが未知数

## 推奨: G1「高校野球監督シミュレーション」(RPG的育成・周回要素を融合)

### 推奨理由

1. **需要の空白が今まさに開いている**: 公式(栄冠クロス)が2026年3月に撤退し、スマホの監督シム本命が不在。検索需要は恒常的で、サ終難民の受け皿がない
2. **個人開発での収益実証がある**: 同ジャンル・同規模の先行事例が累計1,290万円・最高月商257万円(課金76%:広告24%)を公開しており、再現対象として最も確度が高い
3. **バイブコーディング適性が最高**: テキスト/UIベースの監督シムはゲームエンジン不要でReact/Next.jsで作れる。AIの学習データが最も厚い領域
4. **法規制を構造的に回避できる課金設計**: 通貨なし・ガチャなしの直接販売型(殿堂チケット・広告除去・エディット拡張)なら資金決済法・景表法リスクが最小
5. **運営が軽い**: クライアント完結設計+恒常コンテンツ型なら1人で維持可能(対人要素は後回しにできる)
6. **G2(ローグライト采配)の要素は設計として取り込める**: 「3年周回=ラン」「選手世代=ビルド」という栄冠ナイン自体が持つローグライト構造を強調し、RPG的な育成の手応えを出す

### 収益目標(現実ライン)

- 先行事例の課金率5〜6%・課金単価3,000円を参考に、1万DL・DAU1,000で**月10〜30万円**(広告+課金)を1年目の目標に設定
- 甲子園シーズン(3月センバツ・8月選手権)のスパイクを取りに行くリリース計画: **2027年3月のセンバツまでにモバイル版を出せると理想**(それまでWeb版でテストと改善)

### 主要リスクと対策

| リスク | 対策 |
|---|---|
| 「甲子園」商標(阪神電鉄) | タイトル・ゲーム内呼称は「全国大会」「聖地」等に言い換え。実在校名・選手名も不使用 |
| 試合シミュレーションのバランス(先行作の不満点) | 乱数の寄与を「見える化」(能力差が結果に効く設計)し、テストプレイを重ねる |
| 埋もれる(発見性) | Web版を無料公開して野球系SNS/YouTube実況で認知形成→ストア版へ誘導。あるあるネタでシェア導線 |
| 未成年課金の返金 | 課金前の生年月日確認+月額上限+プラットフォームIAPの返金フローに従う |

### MVPスコープ案(次フェーズで詰める)

1. 新規ゲーム開始: 学校設定(校名エディット)・初期部員の自動生成(架空名)
2. 月次ターン制の練習采配(練習メニュー配分・イベント)
3. 大会シミュレーション(地方大会→全国大会、テキスト実況+スコアボード)
4. 選手の成長・卒業・世代交代(3年周回ループ)
5. 殿堂(OB名鑑)= 課金ポイントの土台(MVP時点では無料で枠制限)
6. セーブ(ローカル+クラウド)、スマホ最適化UI(PWA)
7. 収益化はWeb版で検証後にモバイル版(Capacitor+ストアIAP)で本格導入
